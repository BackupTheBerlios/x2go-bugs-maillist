The occurring error refers to a mismatch of SSH host keys.

If you are using SSH Proxy you probably connect to [localhost]:<port>.  
If you have used <port> for another target machine already (i.e. in  
another session profile using SSH Proxy) then this error will occur.

I will leave this bug open, as it needs some GUI guidance in case such  
a situation happens to the user.
