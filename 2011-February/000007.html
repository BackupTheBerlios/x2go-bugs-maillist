mike.gabriel at das-netzwerkteam.de (2011-02-02 13:35) hat geschrieben:

By the time the ~/.x2go/ directory will be flooded with old session  
(cache) directories. The planned behaviour for handling of session  
files will be:

   o place a ,,profile_name'' file into ~/.x2go/<session_name>/ that  
contains the session profile
     name a session was initialized from

Then during runtime:

   o _no_ session cache directories will be removed in debug mode
   o session cache directories will be removed after ,,terminate''  
action directly
   o after connect to a session profile a session list is retrieved  
from the server
        - if a session name is still available (suspended/running) on  
the server -> keep the dir
        - if a session name is not available anymore on the server ->  
remove dir from client
