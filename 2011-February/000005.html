schulz at math.uni-goettingen.de (2011-02-02 11:33) wrote:

We use Kerberos here. For Kerberos to work correctly it is important,  
that the env-variable KRB5CCNAME is set when starting the session, but  
there is no direct way to do that in x2go. the kerberos-ticket-cache  
is created in x2go (via executing pam-stack from ssh-login) but the  
environment-variables are not forwarded to the executed command. I  
made a quick and dirty hack to accomplish this here: i simply edited  
x2goruncommand to set  KRB5CCNAME, but perhaps this should be made  
more cleaner
