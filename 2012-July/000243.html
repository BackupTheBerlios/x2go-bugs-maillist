gschnabel at gmx.de (2012-07-17 19:42) hat geschrieben:

System:
Debian Testing/wheezy with
* x2goserver_3.1.1.4-0~x2go1+wheezy~heuler~20120713~573~build1
* x2goserver-extensions_3.1.1.4-0~x2go1+wheezy~heuler~20120713~573~build1


Problem:
Connections from a Windows XP system to the Debian system with  
X2GoClient V. 3.99.2.2 (Qt - 4.8.1)
http://code.x2go.org/releases/binary-win32/x2goclient/x2goclient-3.99.2.1-setup.exe
are not possible, error popup: "Sessions limit reached:". See attached  
screenshot


Workaround:
Downgrade to
* x2goserver_3.1.1.3-0~x2go1+wheezy~heuler~20120629~567~build1
* x2goserver-extensions_3.1.1.3-0~x2go1+wheezy~heuler~20120629~567~build1

Connections with the older packages work with no problems.

Bug is introduced with commit  
http://code.x2go.org/gitweb?p=x2goserver.git;a=commit;h=d70b136ab99623678f34c12f2175570a856eaa77

Reversal of changes on /usr/bin/x2gosessionlimit make connections work again.
